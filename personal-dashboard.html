USER-CREATED TRIP

多人功能 申請步驟（Supabase）

① LINE Developers（取 LIFF ID）

  1. 前往 LINE Developers Console 用 LINE 帳號登入
  2. 建立 Provider（例：「個人工作室」）
  3. 在 Provider 下建立新 Channel，類型選 LINE Login
  4. 進入 Channel → LIFF 頁籤 → Add 一個 LIFF App
    • Endpoint URL：你的網頁網址（trycloudflare 或部署後的網址）
    • Size：Full
    • Scopes：勾 profile、openid、chat_message.write
    • Bot link：Off
  5. 建立後拿到 LIFF ID（格式像 1234567890-abcdEFGH）

② Supabase（取 URL + Anon Key）

  1. 前往 Supabase Dashboard，登入既有帳號（你已有）
  2. 建立新專案（例：「kevin-travel」），選離你近的 Region（東京/新加坡）
  3. 進專案 → 左側 Settings ⚙ → API
  4. 複製 Project URL（https://xxx.supabase.co）→ 貼到上方 Supabase URL 欄
  5. 複製 anon / public key（很長一串 JWT）→ 貼到 Anon Key 欄
  6. 不要用 service_role key（那是後端用，會洩漏）

③ 建立資料表（執行 SQL）

在 Supabase 專案左側 → SQL Editor → New query，貼上下方 SQL → Run。

-- 同團成員表（presence + 身份）
create table if not exists party_members (
  group_id     text not null,
  user_id      text not null,
  display_name text,
  picture_url  text,
  platform     text,
  last_seen    timestamptz default now(),
  primary key (group_id, user_id)
);
create index if not exists idx_party_members_lastseen
  on party_members (group_id, last_seen desc);

-- 同團聊天訊息（Step 3 會用）
create table if not exists party_messages (
  id          bigserial primary key,
  group_id    text not null,
  user_id     text not null,
  display_name text,
  content     text not null,
  msg_type    text default 'chat',  -- chat / broadcast / groupbuy / system
  payload     jsonb,
  created_at  timestamptz default now()
);
create index if not exists idx_party_messages_group_created
  on party_messages (group_id, created_at desc);

-- 即時位置共享（Step 3 會用）
create table if not exists party_locations (
  group_id   text not null,
  user_id    text not null,
  latitude   double precision not null,
  longitude  double precision not null,
  accuracy   real,
  updated_at timestamptz default now(),
  primary key (group_id, user_id)
);

-- 啟用 Row Level Security
alter table party_members   enable row level security;
alter table party_messages  enable row level security;
alter table party_locations enable row level security;

-- 簡化版策略：anon 可讀寫（之後要加群組代碼驗證）
create policy "anon read members"  on party_members   for select using (true);
create policy "anon write members" on party_members   for insert with check (true);
create policy "anon update members" on party_members  for update using (true);
create policy "anon read messages"  on party_messages  for select using (true);
create policy "anon write messages" on party_messages  for insert with check (true);
create policy "anon read locations" on party_locations for select using (true);
create policy "anon upsert locations" on party_locations for insert with check (true);
create policy "anon update locations" on party_locations for update using (true);

-- 啟用 Realtime（讓前端即時訂閱）
alter publication supabase_realtime add table party_members;
alter publication supabase_realtime add table party_messages;
alter publication supabase_realtime add table party_locations;

④ 群組代碼

自己取一個英文/數字代碼（例：jiuzhai2026）。同團成員填一樣的代碼才會在同一個房間。

⑤ 同團成員怎麼加入

  1. 把網頁網址（要是 LIFF 對應的 endpoint URL）分享給團員
  2. 他們從 LINE 點開 → 自動帶入 LINE 身份
  3. 填相同的 LIFF ID / Supabase URL / Anon Key / 群組代碼
  4. 提示：可以把這四個值放在分享連結的 URL hash，省去團員手動輸入（之後可做）
⚠ 隱私 & 安全：
  • 位置共享預設關閉（Step 3 完成後在「設定」內主動開啟才會回報）
  • 上方 SQL 的 RLS 策略是「anon 可讀寫」最寬鬆版，方便測試。實際運作應改為「只允許自己 user_id 寫」（之後可升級）
  • Anon Key 不是密碼，可公開（但別把網頁公開到 GitHub public repo）
  • Supabase Free Plan：500MB 儲存 + 2GB 頻寬 + 50K MAU，對 20 人小團綽綽有餘
